# 页面原型说明｜MVP v1.0

交互式低保真原型已在当前任务的可视化区域中呈现。它用于确认信息层级和页面流，不代表最终视觉稿。整体视觉采用“留白 + 温暖画材色”的小红书式卡片感，避免过度儿童化，保证老师端的效率。

## 通用导航

| 端 | 导航 | 移动端表现 |
| --- | --- | --- |
| 管理端 | Dashboard、课程、学生、作品、AI 助手；右上角个人菜单 | 折叠为顶部菜单或底部主要入口 |
| 学生/家长端 | 首页、课程、上传、成长；个人页 | 固定底部 4 项导航，上传为高强调入口 |

## 关键状态

| 场景 | 状态与反馈 |
| --- | --- |
| 空列表 | 引导文案 + 单一主按钮，例如“新建第一门课程”。 |
| 图片上传中 | 显示本地缩略图、进度和取消；禁止重复提交。 |
| AI 生成中 | 显示“正在整理鼓励与建议”，保留页面上下文；可取消。 |
| AI 失败 | 清楚提示可重试；不暴露服务商密钥或内部报错。 |
| AI 草稿 | 显著标记“待教师确认”；学生端不可见。 |
| 删除 | 二次确认，说明会影响的记录；采用软删除。 |

## 移动端原则

- 所有主要操作在 44px 可点击区域内完成。
- 表格在手机端改为信息卡，展示头像、姓名/标题、关键状态与更多菜单。
- 上传只需三步：选择图片 → 填写名称和想法 → 提交。
- 作品图片保持原始比例，使用压缩预览，避免全屏图片造成误操作。
