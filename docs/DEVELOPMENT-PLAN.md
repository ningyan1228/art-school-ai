# 开发任务拆解｜MVP v1.0

## 估算口径

以 1 名全栈开发者、已有 GitHub 与服务器账号为前提。工作量是“人日”估算，不含用户验收等待时间。每个阶段完成后都应部署测试环境并验收，再进入下一阶段。

## 里程碑

| 阶段 | 目标 | 预估 | 产出 |
| --- | --- | --- | --- |
| M0 | 项目骨架与部署基线 | 2–3 人日 | Vue/Fastify 项目、环境变量、CI、测试域名 |
| M1 | 可登录、可切换角色 | 3–4 人日 | 登录、JWT、权限守卫、管理/学生首页 |
| M2 | 教学数据闭环 | 6–8 人日 | 课程、学生、作品、上传、成长档案 |
| M3 | AI 作品点评 | 4–5 人日 | SiliconFlow 代理、结构化点评、人工确认 |
| M4 | AI 教案/家长反馈与 PWA | 3–4 人日 | 两项生成工具、安装体验、离线壳页 |
| M5 | 测试、试运行与上线 | 3–5 人日 | 安全检查、备份、监控、使用说明 |

总计：约 21–29 人日。

## M0：项目初始化与部署基线（P0）

- [ ] 创建单仓库目录：`frontend/`、`backend/`、`docs/`。
- [ ] 初始化 Vue 3 + Vite + TypeScript，配置 Tailwind、Pinia、Vue Router、ESLint/Prettier。
- [ ] 初始化 Fastify API，增加 health check、统一错误格式、日志与环境变量校验。
- [ ] 增加 SQLite migrations、种子数据脚本和本地 `.env.example`。
- [ ] 配置前端 GitHub Actions 构建并发布 GitHub Pages。
- [ ] 服务器 Docker Compose：API、Nginx、持久卷（SQLite 与 uploads），配置 HTTPS 和 CORS。
- [ ] 验收：前端可访问；`/health` 返回成功；前端可安全请求测试 API。

## M1：身份与基础界面（P0）

- [ ] 迁移 `users`、`students` 与初始管理员账户。
- [ ] 后端：登录、JWT 鉴权、中间件、角色权限、获取当前用户。
- [ ] 前端：登录页、token 持久化、路由守卫、退出登录。
- [ ] 管理端：壳布局、侧边导航、Dashboard 的真实统计接口。
- [ ] 学生端：底部导航、首页课程/任务/最近作品占位真实接口。
- [ ] 验收：四种角色不能越权读取他人资源；刷新页面登录仍有效；禁用账户不能登录。

## M2：课程、学生、作品闭环（P0）

### 课程与学生

- [ ] 后端：`courses`、`lessons`、`enrollments` 的 CRUD 与权限检查。
- [ ] 前端：课程列表、创建/编辑抽屉或页面、章节排序。
- [ ] 后端：学生档案、家长绑定、列表搜索/分页。
- [ ] 前端：学生列表、学生档案、报名课程显示。

### 上传与成长档案

- [ ] 后端：multipart 图片校验、压缩/缩略图策略、随机文件名、上传接口。
- [ ] 后端：作品 CRUD、作品归属权限、软删除与图片清理队列。
- [ ] 前端：教师上传作品、学生上传作品、进度/错误状态。
- [ ] 前端：作品列表/详情、学生成长时间轴、课程进度展示。
- [ ] 验收：学生只能上传并查看自己的作品；教师只能操作被授权学生；删除作品不会立刻误删原文件。

## M3：AI 作品点评（P0）

- [ ] 选定 SiliconFlow 模型与版本，写入服务端环境变量；建立超时、重试与限流策略。
- [ ] 编写受版本控制的点评提示词和 JSON schema 校验器。
- [ ] 后端：根据作品 URL/二进制调用 SiliconFlow，多模态能力不足时输出明确降级提示。
- [ ] 后端：创建 `ai_requests` 审计日志和 `ai_reviews` 草稿；不得把密钥、完整上游错误给客户端。
- [ ] 前端：点评生成页、加载/失败/重试、可编辑结构化结果。
- [ ] 前端：教师确认公开、拒绝/重生成；学生端只展示 confirmed 内容。
- [ ] 验收：用 10 张不同作品测试，结果可解析、语气友好、无评分/贬低表述；权限与错误处理通过。

## M4：教学助手与 PWA（P1）

- [ ] 后端与前端：教案生成输入、结构化结果、保存草稿。
- [ ] 后端与前端：家长反馈草稿生成、编辑、复制。
- [ ] 配置 Web App Manifest、图标、service worker 与更新提示。
- [ ] 核查离线策略：仅缓存静态壳和公开资源；退出登录后不保留私密数据。
- [ ] 验收：Android Chrome 可安装；弱网下已加载页面可打开壳页；AI 工具的草稿可编辑保存。

## M5：质量、运维和上线（P0）

- [ ] 单元测试：鉴权、权限、AI JSON 解析、图片类型/大小校验。
- [ ] API 集成测试：登录、CRUD、跨角色越权、上传、AI 草稿确认。
- [ ] E2E 测试：管理员建课 → 建学生 → 上传作品 → AI 点评确认 → 学生查看成长记录。
- [ ] 性能与安全：Lighthouse 移动端检查、CORS 白名单、速率限制、HTTP 安全头、依赖漏洞扫描。
- [ ] 运维：SQLite 备份/恢复演练、日志轮转、磁盘空间/接口错误监控、密钥轮换流程。
- [ ] 上线：隐私告知与家长授权、试用机构培训、反馈收集表。

## 建议的开发顺序（首个可演示版本）

1. 做 M0 和 M1，先确认登录、角色切换及两端布局。
2. 完成课程 + 学生 + 作品上传，形成没有 AI 也能使用的最小闭环。
3. 接入 AI 点评，强制保留教师确认节点。
4. 最后再做教案、家长反馈和 PWA 优化。

## 需要你在编码前确认的事项

1. 产品名称、Logo/主色与正式域名（可先使用临时名称）。
2. 首批机构是单校区还是要按机构隔离数据；本 PRD 按单机构 MVP 设计。
3. SiliconFlow 要使用的具体模型，以及其是否提供你选择的图片理解接口。
4. 服务器域名和 HTTPS 证书是否已具备；图片是否允许存放在自有服务器。
5. 是否需要一开始就让家长注册；建议 MVP 首发先开管理员、教师、学生账号，家长端作为可开关功能。
